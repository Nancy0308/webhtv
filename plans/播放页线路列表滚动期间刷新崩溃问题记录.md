# 播放页线路列表滚动期间刷新崩溃问题记录

## 基本信息

- 记录日期：2026-07-16
- 应用版本：5.5.6
- 设备：MediaTek LED Projector
- 系统：Android 11（SDK 30）
- 当前状态：已修复，待真机长期回归
- 严重程度：P2（中等）；若普通遥控操作可稳定复现，应提升为 P1

## 问题现象

播放页面切换线路或来源时，应用弹出错误信息并中断当前操作。核心异常为：

```text
java.lang.IllegalStateException:
Cannot call this method while RecyclerView is computing a layout or scrolling
```

异常对象为播放页中 ID 为 `app:id/flag` 的 Leanback `HorizontalGridView`，即横向线路/来源列表。

## 根因分析

这是 RecyclerView 列表刷新与布局/滚动过程发生冲突导致的 UI 时序崩溃，不属于网络、视频源、播放器解码器或设备数据损坏问题。

调用过程如下：

1. 用户在线路/来源列表中选择一个 `Flag` 项目。
2. `VideoActivity.onItemClick(Flag)` 调用 `mFlagAdapter.setSelected(item)`。
3. `FlagAdapter.setSelected()` 同步调用 `notifyItemRangeChanged(0, getItemCount())` 刷新整行。
4. 此时 `HorizontalGridView` 仍可能正在处理焦点滚动或计算布局。
5. RecyclerView 的 `assertNotInLayoutOrScroll()` 检测到非法刷新并抛出 `IllegalStateException`。

相关代码位置：

- `app/src/leanback/java/com/fongmi/android/tv/ui/activity/VideoActivity.java`：`onItemClick(Flag)`
- `app/src/leanback/java/com/fongmi/android/tv/ui/adapter/FlagAdapter.java`：`setSelected()` 中同步调用 `notifyItemRangeChanged()`
- `app/src/leanback/java/com/fongmi/android/tv/ui/base/BaseActivity.java`：随后还会通过 `post()` 执行一次 `notifyDataSetChanged()`，存在重复刷新

## 可能的触发条件

- 遥控器快速左右移动后立即按确定键。
- 线路列表仍在平滑滚动时切换来源。
- 播放页面刚显示，焦点或列表仍在重新布局时操作。
- 低性能电视或投影设备布局耗时较长，触发概率更高。

MediaTek 投影设备可能更容易暴露该问题，但设备不是根本原因。

## 影响评估

- 会中断用户当前播放操作并显示错误信息。
- 用户可能需要关闭错误提示、重新选择线路或重新进入播放页。
- 不会损坏配置、历史记录或媒体数据。
- 影响范围主要是 TV/Leanback 播放页的线路/来源横向列表。
- 当前判断为偶发时序问题；若在常规遥控操作下可高概率复现，则属于高优先级崩溃问题。

## 后续修复方向

修复时应重点考虑：

1. 避免在 RecyclerView 正在计算布局或滚动时同步调用 Adapter 的刷新方法。
2. 将刷新延迟到 View 消息队列的下一轮，或确认列表不处于 `isComputingLayout()` 状态后再执行。
3. 仅刷新旧选中项和新选中项，避免每次刷新整个线路列表。
4. 清理 `setSelected()` 内同步刷新与 `VideoActivity` 后续异步全量刷新之间的重复通知。
5. 使用低性能 Android 11 TV/投影设备进行快速方向键加确定键的压力回归测试。

## 修复实现

本次修复采用以下策略：

1. `FlagAdapter.setSelected()` 只修改线路选中状态，不再同步调用 `notifyItemRangeChanged()`。
2. 选择线路前后分别记录旧选中位置和新选中位置。
3. 通过 `BaseActivity.notifyItemsChanged()` 延迟刷新旧、新两个条目，不再重复全量刷新整行。
4. 公共 RecyclerView 刷新助手在执行通知前检查 `isComputingLayout()`；如果列表仍在布局或滚动计算中，则延迟到下一帧重试。
5. 对无效位置、重复位置和列表已变化后的越界位置进行过滤。

这样可以保证业务选中状态立即完成，同时把 Adapter 通知移出 RecyclerView 的布局/滚动调用栈，避免再次触发 `assertNotInLayoutOrScroll()`。

## 建议验证场景

- 在线路列表中连续快速左右移动焦点并按确定键。
- 列表平滑滚动尚未停止时连续切换线路。
- 播放页首次加载完成前后立即操作线路列表。
- 分别在单线路、多线路和线路数量较多的媒体中测试。
- 在 Android 11 MediaTek 设备以及性能较好的 TV 设备上对比测试。
